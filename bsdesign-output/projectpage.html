Project Idea

The Rust Helper is a free, open-source application built using Vue, meant as an assistant for the video game Rust. The survival videogame Rust by facepunch studios requires keeping track of a variety of information for optimal play. This application is meant to aid in this endeavor, primarily for solo players and small groups who are new to the game. It is a web-application hosted on GitHub pages and can be reached through this link. In the following sections, I will briefly showcase some of the features, technologies, and challenges that were involved in creating this software.

Technology Stack

The Rust Helper application uses Vue to fully control the view. Additionally, a variety of other packages are used. A VueX store is used to provide a centralized state between the different components. BootstrapVue is used for easy styling. v-select is used for more visually pleasing select boxes. fabric.js is used to display the map and additional markers which can be manipulated. Moment.js is used for easy display of timestamps. Finally, VueX persisted state is used to store the VueX state in the LocalStorage, in order to maintain the state between sessions.

Functionality

The Rust Helper is built as a modular software, and development happened on each module at a time.

Server Selector
Allows the addition of differently named servers. This is the base module that deals with the creation, deletion and selection of different servers, which all store custom information for each of the submodules.

Furnace Calculator
Generates the output to any given furnace input using a “breakpoint” based algorithm (allowing the computation of many steps in one function iteration, rather than a single furnace tick). Supports setting timers for the expiration of the furnaces, and dealing with multiple furnaces per server and across servers.

Map View
Uses the RustMaps API to generate map images for Rust Servers, given a seed and map size. Monument positions and names are loaded directly from the RustMaps API and are displayed. Offers the option to add various shape and color markers, which can be moved, rescaled, and rotated.

Build Cost Calculator
Allows the summation of costs for common building materials, tools, and deployables, creating a sort-of “shopping list” for base materials and their costs required.

Lessons Learned

While most of the development of this application went relatively smooth, there are still some things I would do different the next time around.

The first module, the furnace calculator, was for the most part written without the use of VueX, by simply using the intended Vue design of parent-children communication. While this worked well as a modular approach, for the application as a whole, implementing VueX was definitely the correct choice. While this may sound a bit “acedemic”, there is truly a design tradeoff to be made here between convenient and centralized state access, and a clear hierarchy and data flow.

Another problem I had to deal with was trying to find an appropriate package for easy canvas operations (such as movable objects). At first I considered marker.js, however, it felt very inconvenient to implement and the getting started documentation did not make much sense. After some further search, I settled on fabric.js. Fabric.js has been very nice and easy to implement and offers a variety of options for live marker manipulation. I can definitely recommend this tool for anyone looking to work with the canvas on a high level. While I intentionally kept the map view module rather basic to finalize development, I am confident that fabric.js would have been the right choice for a more detailed implementation. In the future, I will likely research and try out my intended packages in a sandbox before using them in the final project, to make sure they suit the case at hand.

Overall, I also think I took away a lot of knowledge about Vue that I will be able to use in the future to create easy dynamic frontend views, either for a full-scale application or just an embedded part of a webpage.
